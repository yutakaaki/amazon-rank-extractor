# Amazon売れ筋ランキング 書籍情報抽出ツール

Amazonの公開ランキングでは見られない **201位〜1000位** の書籍を対象に、
各商品ページの登録情報に記載された「Amazon売れ筋ランキング」の数値を読み取り、
指定した順位範囲・ジャンルの書籍情報（タイトル / 巻数 / 発売日）を抽出します。

## iPad など他端末から使う（同じWi-Fi内）＋ 毎朝の自動抽出

ターミナルを使わず、ブラウザの操作画面から使えます。

### セットアップ（Macで一度だけ）
1. Finderで `amazon-rank-extractor/scripts` を開く
2. **`install.command` をダブルクリック**（初回は右クリック→「開く」。pmset設定で管理者パスワードを求められます）

これで以下がインストールされます。
- **Web操作画面の常駐**：Macログイン中は常に起動（落ちても自動再起動）
- **1日2回の自動抽出（6:00 / 18:00）**：本日発売のコミックを自動取得して `results/` に保存
- **朝→夕のランク変化比較**：18:00の回で、同じ本の `6時順位 → 18時順位` を突き合わせ、
  **ランク上昇率(%)** 付きの比較CSV（`..._朝夜比較.csv`）を生成
- **スリープからの自動起床**：6:00・18:00の直前にMacが自動で起き、抽出後また眠る
  （**電源アダプタ接続が必須**。18:00分の起床のため `pmset` の限定sudo許可を設定）

### 朝→夕 比較CSVの見方
`6:00`＝予約購入までの規模、`18:00`＝予約＋当日購入の規模の目安。
同一書籍（ASIN一致）でランクが両方取れたものだけを対象にし（圏外は無視）、
列は `タイトル / 巻数 / 発売日 / 6時順位 / 18時順位 / 順位変化 / ランク上昇率(%)`。
- **ランク上昇率(%)** = (6時順位 − 18時順位) ÷ 6時順位 × 100。プラスほど当日よく売れた指標。
- 18時順位が良い順（上位）に並ぶので、**上位20件ほど**が「発売日にどれだけ伸びたか」を見やすい。

### 使い方
- iPad等（同じWi-Fi）のブラウザで `http://<MacのIP>:3000` を開く（IPはインストール時に表示）
- 画面で「本日発売／順位帯」を選んで「抽出を開始」→ 進捗が出て、完了すると表で表示
- 「保存済みの結果」から過去の自動抽出も閲覧・CSVダウンロード可
- 自動抽出の結果も同じ画面の一覧に出ます

### 解除
`scripts/uninstall.command` をダブルクリック（常駐・自動抽出・自動起床をすべて停止）。

### 注意（スリープ自動実行）
- **電源アダプタに接続**しておくこと（バッテリー駆動だと指定時刻の自動起床が効きません）。フタは閉じていてもOK。
- Macを**シャットダウン**していると動きません（スリープはOK）。
- 実行時刻を変えたい場合は `scripts/install.command` の `StartCalendarInterval`（6時/18時）と
  `WAKE_TIME`（朝の起床）、`src/daily.js` の夕方起床予約時刻を編集して再度ダブルクリック。

## 仕組み

201〜1000位はベストセラー一覧（上位約100件）には現れないため、
**「売れ筋ランキング順」でソートしたジャンル全件検索**を候補源にします。

1. ジャンル検索結果を売れ筋順に並べ、指定順位に対応するページのみ取得して候補ASINを収集
   （順位 r ≒ `ceil(r / 16)` ページ目。前後に `PAGE_MARGIN` ページの余裕を取る）
2. 各商品ページを開き、**登録情報内の「Amazon売れ筋ランキング - <ジャンル>」の順位のみ**を参照
3. 指定範囲（例: 201〜400位）に入る本だけを抽出
4. タイトル・巻数・発売日を整形して出力（JSON または CSV）

> 検索順位は「どのページを候補として見るか」の目安にのみ使い、
> 順位の確定は必ず各商品ページの登録情報の値で行います。それ以外のページの順位は使いません。

## セットアップ

Node.js（18以上）が必要です。未インストールなら https://nodejs.org からどうぞ。

```bash
cd amazon-rank-extractor
npm install
npx playwright install chromium
```

## 使い方

### 順位モード（指定順位帯を抽出）
```bash
node src/cli.js --genre コミック --from 201 --to 400 --out comic.csv
```

### 発売日モード（指定日に発売の本を全件抽出）
```bash
node src/cli.js --genre コミック --date 2026-06-17 --out today.csv
node src/cli.js --genre コミック --date --out today.csv   # --date 値なし or 省略で「当日」
```
- 指定日に発売の本を**全件**、タイトル・巻数・順位・発売日で出力します。
- **順位が無い本は「—」**で記録します（その日の発売には圏外の本も多く含まれるため）。
- **未来日（明日以降）は対象外**：予約上の順位になるため弾きます。当日・過去のみ可。
- 発売日順検索を**二分探索**で対象日まで飛ぶので、過去の日付でも高速に到達します。

| オプション | 説明 |
| --- | --- |
| `--genre` | ジャンル（`コミック` / `ライトノベル`。`src/config.js` で追加可） |
| `--from` `--to` | 【順位モード】抽出する順位範囲（201〜1000） |
| `--date` | 【発売日モード】抽出する発売日（`YYYY-MM-DD`。値なし/省略で当日） |
| `--out` | 出力先。`.csv` ならCSV、それ以外はJSON（既定 `result.json`） |
| `--headful` | ブラウザを表示して実行（デバッグ用） |
| `--yes` | 件数上限超過時の確認を飛ばして続行 |

## 安全装置（仕様対応）

- **201位未満は拒否**: `--from` が200以下だと「公開ランキングで見られる」として停止。
- **1000位超は拒否**: 対象は最大1000位。
- **順位数の上限アラート**: 指定範囲が `MAX_RANK_SPAN`（既定300件）を超えると、
  スクレイプ量が多すぎるとして**抽出実行前に**警告して中止します。`src/config.js` で調整可。
- **発売日モードの件数上限**: 該当日の候補が `MAX_DAILY_ITEMS`（既定200件）を超えると、
  商品ページ巡回の前に警告。続行するには `--yes` を付けて再実行します。
- **未来日の拒否**: 発売日モードで明日以降を指定すると停止します。
- 各ページアクセスの間に待機（既定2.5秒）を入れ、過負荷を避けています。

## ジャンルの追加

`src/config.js` の `GENRES` に、ジャンル名と `searchBase` を追記するだけです。

- `label`: 商品ページの「Amazon売れ筋ランキング - <ここ>」と照合する文字列
- `searchBase`: そのジャンルの検索結果を「売れ筋ランキング順」で並べたURL。
  Amazonで対象ジャンルを絞り込み、並び順を「売れ筋ランキング」にした後、
  URLから `ds` / `qid` / `ref` などのセッション固有パラメータを取り除いたもの。

## 注意

- Amazonのレイアウト変更やbot検知により取得できない場合があります。その際は
  `--headful` で挙動を確認し、`src/scraper.js` のセレクタや待機時間を調整してください。
- スクレイピングはAmazonの利用規約の範囲内・自己責任でご利用ください。
